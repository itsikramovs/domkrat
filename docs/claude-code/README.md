## 🚀 С чего начать

### Шаг 1: Прочитайте документацию проекта
Сначала ознакомьтесь с основной документацией (папка выше — `domkrat-docs/`):
1. `01-PROJECT-OVERVIEW.md`
2. `03-ARCHITECTURE.md`
3. `04-DATABASE-SCHEMA.md` + `PART2.md`
4. `05-BUSINESS-FLOWS.md`

### Шаг 2: Прочитайте инструкции для AI
1. **WORKFLOW.md** — как организовать команду и процесс
2. **CLAUDE-CODE-PLAN.md** — план задач для AI
3. **PROMPTS-LIBRARY.md** — готовые промпты

### Шаг 3: Подготовьте окружение
1. Создайте GitHub репозиторий
2. Скопируйте **CLAUDE.md** в корень репозитория
3. Скопируйте всю папку `docs/` в репозиторий
4. Настройте Claude Code (https://docs.anthropic.com/claude/docs/claude-code)
5. Откройте проект в Claude Code

### Шаг 4: Начните разработку
Идите по плану из **CLAUDE-CODE-PLAN.md**:
- **Phase 0**: Bootstrap (делает Tech Lead вручную)
- **Phase 1**: Foundation (auth, users) — первые задачи для Claude
- **Phase 2-6**: Остальной MVP

Для каждой задачи — используйте шаблон промпта из **PROMPTS-LIBRARY.md**.

## 🎯 Ключевые принципы AI-driven разработки

### 1. CLAUDE.md — это священный файл
Этот файл читает Claude автоматически в каждой сессии. В нём:
- Правила безопасности
- Конвенции кода
- Антипаттерны (что НЕ делать)
- Структура модулей

**Всегда обновляйте его** при изменении правил.

### 2. Дробите задачи
Claude отлично работает с **маленькими, конкретными** задачами (~150-300 LOC).
Большие задачи дробите на 3-10 маленьких.

### 3. Тесты — это контракт
Каждая задача должна иметь **тесты**. Без тестов Claude может незаметно сломать что-то при следующей задаче.

### 4. Code review — обязателен
Не мерджите PR без вдумчивого review. Даже зелёные тесты не гарантируют корректность.

### 5. Тех. лид — критичная роль
AI-разработка работает только при наличии **опытного Tech Lead**, который:
- Принимает архитектурные решения
- Декомпозирует задачи
- Проверяет каждый PR
- Корректирует Claude когда нужно

---

## ⚠️ Что НЕ нужно делать

1. **Не давайте Claude задачу типа "сделай всю фичу"** — разбейте.
2. **Не пропускайте code review** — даже если торопитесь.
3. **Не давайте Claude принимать архитектурные решения** — это работа Tech Lead.
4. **Не мерджите без тестов** — это создаст технический долг.
5. **Не работайте без CLAUDE.md** — потеряете контекст между сессиями.
